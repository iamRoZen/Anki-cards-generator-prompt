# 设置
- 语言: 简体中文
- 输入格式: MD格式的归纳讲义（或原始文档）
- 输出: txt代码块

# 任务：直接从讲义制作Anki卡组

阅读提供的MD格式讲义，直接制作Anki卡组，无需先制作问题列表。

## 制卡原则

### 1. 问题设计
1. 从讲义的标题层级提取问题
2. 每个四级标题(####)通常对应2-4张卡片
3. 每个五级标题(#####)通常对应1-3张卡片
4. 问题字数<15字
5. 合并并列内容，不强行拆分简短列表
6. 一个问题原则: 一张卡片只问一个问题, 避免使用"和,或,且,而,与,还,并,伴,又,同,及"（除非比较两个客体）

### 2. 答案要求
1. **字数**：<50字
2. **条目**：每条<15字
3. **醒目并列**: 不同句子中间用①②③④⑤隔开
4. **hook**: 必要, 2-6字
5. **来源**：仅从讲义提取，禁止幻觉
6. **符号化压缩**（见下方规则）

### 3. 格式模板

```html
主题 || 子主题 || 问题 <code>要点数量</code> <span style="color:颜色">[掌握程度]</span>[TAB]答案
````

**分隔符**：问题和答案之间用**单个tab**（不是空格）

**掌握程度标记**：

- `[掌握]` - `color:red`
    
- `[熟悉]` - `color:orange`
    
- `[了解]` - `color:blue`
    

## 答案模板

### A. 小标题结构-多层级

```html
<h3>要点1</h3>
<ol>
  <li>hook: 内容1(→A)</li>
  <li>hook: 内容2(→B)</li>
  <li>hook: 内容3→A</li>
</ol>
<h3>要点2</h3>
<ol>
  <li>hook: 内容</li>
  <li>hook: 内容</li>
</ol>
```

### B. 压缩表格-对比

```html
<table>
  <thead>
    <tr><th>属性/特征</th><th>分类A</th><th>分类B</th><th>分类C</th></tr>
  </thead>
  <tbody>
    <tr><td><b>属性1</b></td><td>①特征1②特征2</td><td>①特征1②特征2</td><td>①特征1②特征2</td></tr>
    <tr><td><b>属性2</b></td><td>特征1→特征2</td><td>特征1→特征2</td><td>特征1→特征2</td></tr>
  </tbody>
</table>
```

### C. 精简列表-列举

```html
<ol>
  <li><b>hook：</b>核心内容(→结果)</li>
  <li><b>hook：</b>核心内容(→结果)</li>
</ol>
```

```html
<ol>
  <li>hook: ①A1→A1'; ②A2→A2'</li>
  <li>hook: ①B1→B1'; ②B2→B2'</li>
</ol>
```

## 符号化压缩规则

### 1. 标准符号

| 符号    | 含义       | 示例        |
| ----- | -------- | --------- |
| →     | 发展/导致/因果 | 炎症→纤维化    |
| ↑↓    | 增减       | 眼压↑、视力↓   |
| ①②③④⑤ | 并列       | ①眼痛②头痛③恶心 |
| /     | 选择       | 细菌性/病毒性   |
| ±     | 可能       | 发热±畏寒     |

### 2. 缩写

|原词|缩写|原词|缩写|
|---|---|---|---|
|维生素A|VitA|抗生素|Abx|
|手术治疗|手术|药物治疗|药物|
|临床表现|症状|并发症|并发|

### 3. 删除冗余

**删除词汇：**

- 助词：的、地、得、了、着、过
    
- 连词：并且、以及、同时、另外
    
- 修饰：具有、进行、表现为、可以
    

**保留要素：**

- 核心名词、动词、形容词、药物、手术
    
- 数值、部位、程度词
    
- 因果关系词
    

## 掌握程度判断

根据讲义中知识的重要性判断：

- **[掌握]**：诊断标准、治疗原则、病因分类、主要临床表现、关键数值
    
- **[熟悉]**：次要表现、辅助诊断、常见并发症、次要病因
    
- **[了解]**：历史背景、少见情况、补充知识、流行病学数据
    

## 输出要求

**严格按照以下格式输出：**

1. 用```代码块包裹所有输出
    
2. 问题和答案之间用**tab**分隔（不是空格）
    
3. 每行一个卡片，行间不空行
    
4. 按讲义顺序逐节制作
    

```
主题 || 子主题 || 问题<code>数量</code> <span style="color:颜色">[程度]</span>[TAB]答案
主题 || 子主题 || 问题<code>数量</code> <span style="color:颜色">[程度]</span>[TAB]答案
```

**其中[TAB]表示tab制表符**
