# Anki 制卡 Prompt

> 从讲义中提取高价值考点，制作原子化、可测试的Anki卡片。

## 适用场景

- 将课程讲义转为Anki卡组
- 需要结构化、分类明确的记忆卡片
- 适合ADHD用户的简洁格式

# 设置

- 语言：简体中文
- 输入：MD格式讲义（或原始文档）
- 输出：txt代码块（可直接导入Anki）

# 核心原则

1. **宁拆勿合**：知识点超过50字就拆分成多张卡片
2. **宁缺毋滥**：只用讲义内容，绝不臆测补充
3. **序号强迫症**：2个以上并列项必须用①②③，严禁顿号、逗号、斜杠
4. **质量标准**：每张卡片必须做到"看到问题能立即联想答案"

# 输出格式

## 整体结构

每行4个字段，用**TAB**分隔：

```
问题字段[TAB]答案[TAB]理解记忆[TAB]知识类型
```

用 ``` ```代码块包裹所有输出，每行一张卡片，行间不空行。

---

# 字段1：问题

## 格式

```
主题 || 子主题 || 问题 <code>要点数</code> <span style="color:颜色">[掌握程度]</span>
```

## 掌握程度

| 程度 | 颜色 | 适用 |
|:---:|:---:|:---|
| [掌握] | `color:red` | 核心概念、基本原理、必考内容 |
| [熟悉] | `color:orange` | 次要概念、常见应用 |
| [了解] | `color:blue` | 背景知识、扩展内容 |

## 问题设计原则

**基本要求**：
- 字数<15字
- 一张卡片只问一个问题
- 避免"和、或、且、及"连接多个问题（除非比较两个客体）

**场景化提问**（优先使用）：

| 类型 | 示例 |
|:---|:---|
| 决策导向 | 哪些情况下应使用XX？什么时候选择XX？ |
| 应用导向 | XX如何使用？XX适用于哪些场景？ |
| 识别导向 | 如何识别XX？如何区分X和Y？ |
| 因果导向 | 为什么XX会导致YY？XX的原理？ |
| 数值导向 | XX的判断标准？XX的阈值？ |

**避免空洞提问**：

| ❌ 避免 | ✅ 改为 |
|:---|:---|
| XX的特点？ | XX如何使用？/ XX适用于？ |
| XX的定义？ | 如何判断是XX？ |
| 情况1/2/3？ | 轻度处理/重度处理/紧急处理？ |
| XX机制？ | XX如何导致YY？ |

# 字段2：答案

## 基本要求

- **总字数**：<50字
- **每条**：<15字
- **来源**：仅从讲义提取，禁止幻觉
- **hook**：必须有，2-6字，用`<b>`加粗
- **重点**：用`<u>`下划线标注

## 序号与列表规则（核心！）

**①②③④⑤** = AND（都要记）
**⑴⑵⑶⑷⑸** = OR（分类/选择）

**严禁**：用 `、` `，` `+` `/` 连接多个需记忆的要点

## 格式选择

```
问：答案有多个并列项吗？
├─ 否 → 直接写文字
└─ 是 → 问：每个li内部还有嵌套并列吗？
         ├─ 否 → 用<ol><li>有序列表
         └─ 是 → li内部用①②③序号

问：内容需要横向对比吗？（如不同类型/阶段/方法）
├─ 是 → 用<table>表格
└─ 否 → 用列表
```

| 场景 | 格式 | 示例 |
|:---|:---|:---|
| 简单并列（无嵌套） | `<ol><li>` | `<ol><li>变量</li><li>函数</li><li>类</li></ol>` |
| 带hook分类 | `<ol><li><b>` | `<ol><li><b>输入：</b>键盘/文件</li></ol>` |
| 嵌套并列 | li内用①②③ | `<li><b>前提：</b>需要①安装环境②配置路径③导入模块</li>` |
| 叙述中突然并列 | ①②③ | `需要①配置②初始化③调用` |
| 易混淆对比 | `<table>` | 见示例 |

## 格式示例

**简单并列**：
```html
<ol><li>变量声明</li><li>函数定义</li><li>类创建</li></ol>
```

**带hook分类**：
```html
<ol><li><b>输入：</b>键盘/文件</li><li><b>输出：</b>屏幕/文件</li></ol>
```

**嵌套并列**：
```html
<ol><li><b>前提：</b>需要①安装环境②配置路径③导入模块</li><li><b>步骤：</b>①初始化②调用③释放</li></ol>
```

**对比表格**：
```html
<table><thead><tr><th>类型</th><th>特点</th><th>适用</th></tr></thead><tbody><tr><td><b>数组</b></td><td>连续存储</td><td>快速访问</td></tr><tr><td><b>链表</b></td><td>离散存储</td><td>频繁增删</td></tr></tbody></table>
```

**错误示例**：
```html
❌ ①变量②函数③类  （简单并列不该用序号，用<ol><li>）
❌ 包含头部、主体、尾部  （嵌套并列不该用顿号）
❌ 数组连续存储快速访问，链表离散存储频繁增删  （对比内容不该堆一起）
```

# 字段3：理解记忆

**要求**：<30字，帮助建立知识联结

| 类型 | 适用场景 | 示例 |
|:---|:---|:---|
| **助记符** | 多项并列需记忆顺序 | "请灰太狼滚"=①请求②灰度③太空④狼群⑤滚动 |
| **类比** | 抽象概念需具象化 | 栈像叠盘子，后放先取(LIFO) |
| **联想** | 需连接已有知识 | 类似XX原理，只是YY不同 |
| **词源** | 专业术语难记 | hyper(过度)+tension(张力)=高血压 |
| **对比** | 易混淆概念 | vs XX：关键区别在于YY |
| **因果链** | 需理解原理 | A→B→C，因为XX所以YY |
| **口诀** | 步骤/流程类 | "一看二听三检查" |
| **画面** | 需视觉化记忆 | 想象XX场景，YY在ZZ |
| **数字规律** | 数值类知识 | 记住基准值XX，其他按倍数推 |

**设计原则**：
- 简短有效，一句话点透
- 用谐音、双关、夸张增强记忆
- 尽量关联已知知识
- 实在没有好助记时，用"关键词：XX"提取核心

# 字段4：知识类型

| 类型 | 标签 | 特征 | 学习策略 |
|:---|:---|:---|:---|
| **原子化** | `原子` | 单一事实/数值/定义；不可再拆；有明确唯一答案 | 助记术+间隔重复 |
| **框架性** | `框架` | 分类/对比/结构/流程；涉及多概念关系 | 先理解结构，再记细节 |
| **应用型** | `应用` | 机制/因果/实际应用；涉及"为什么""怎么用" | 理解因果链，能推导就不死背 |

**判断方法**：
- 单一事实/数值/定义 → 原子
- 分类/对比/结构/流程 → 框架
- 为什么/机制/因果/实际应用 → 应用
- 不确定 → 默认原子

# 符号压缩规则

## 标准符号

| 符号 | 含义 | 示例 |
|:---|:---|:---|
| → | 发展/导致/因果/流程 | 输入→处理→输出 |
| ↑↓ | 增减/升降 | 效率↑、复杂度↓ |
| ①②③④⑤ | AND（都要记） | ①定义②实现③调用 |
| ⑴⑵⑶⑷⑸ | OR（分类/选择） | ⑴方法A⑵方法B |
| ± | 可能/可选 | 参数±默认值 |
| `<u>下划线</u>` | 重点 | `<u>核心概念</u>` |
| `<b>加粗</b>` | hook/分类 | `<b>前提：</b>` |
| ≥≤ | 大于等于/小于等于 | ≥60分 |
| ≈ | 大约/约等于 | ≈100ml |
| ≠ | 不等于 | A≠B |

## 删除冗余词

**删除**：的、地、得、了、着、过、并且、以及、同时、另外、具有、进行、表现为、可以、能够

**保留**：核心名词、动词、形容词、数值、条件、程度词、因果关系词
