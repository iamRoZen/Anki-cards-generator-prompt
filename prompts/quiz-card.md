# 选择题制卡 Prompt

> 从讲义中提取考点，制作带详细解析的选择题Anki卡片。

## 适用场景

- 将课程讲义转为选择题形式的Anki卡组
- 需要通过做题巩固知识
- 备考刷题

## Prompt


# 设置

- 语言：简体中文
- 输入：MD格式讲义（或原始文档）
- 输出：txt代码块（可直接导入Anki）

# 核心原则

1. **题干清晰**：每个题干<30字，问题明确
2. **选项简洁**：每个选项<20字，格式统一
3. **干扰合理**：错误选项来自讲义相关内容，不臆造
4. **解析充分**：正确答案说明理由，错误选项说明排除原因

# 输出格式

每行4个字段，用**TAB**分隔：

```
问题字段[TAB]选项[TAB]答案[TAB]解析
```

用 ``` ```代码块包裹所有输出，每行一题，行间不空行。

---

# 字段1：问题

## 格式

```
主题 || 子主题 || 题干？
```

## 题干设计原则

**基本要求**：
- 字数<30字
- 问题明确，不模糊
- 避免双重否定
- 少用"除外""不包括"等

**场景化提问**（优先）：

| ✅ 推荐 | ❌ 避免 |
|:---|:---|
| 以下哪项是XX的核心特征？ | 以下哪项正确？ |
| 当出现XX情况时，应选择？ | 关于XX的说法，错误的是？ |
| XX的主要作用是？ | XX的特点包括？ |

---

# 字段2：选项

## 格式

选项之间用`<br>`分隔：

```
A选项1<br>B选项2<br>C选项3<br>D选项4
```

## 要求

- 通常4个选项（A/B/C/D）
- 每个选项<20字
- 格式统一（都是短语或都是完整句）
- 长度尽量接近

## 干扰项设计

**合理来源**：
- 讲义中相关但不正确的内容
- 容易混淆的概念
- 其他场景下的正确答案
- 数值相近但不准确的选项

**避免**：
- 完全臆造的内容
- 明显错误的选项（送分）
- 过于刁钻的选项
- 与题干无关的内容

---

# 字段3：答案

- 单选/情境/配对：单个字母，如 `A`
- 多选：多个字母，如 `ABD`

---

# 字段4：解析

## 基本要求

- 字数：80-150字
- 分段显示：用`<br><br>`分隔不同部分
- 颜色标注：区分正确/错误/记忆要点

## 颜色方案

| 模块 | 颜色代码 | 用途 |
|:---|:---|:---|
| ✓ 正确答案 | `#2E7D32`（深绿） | 说明为什么对 |
| ✗ 错误选项 | `#D32F2F`（深红） | 说明为什么错 |
| 💡 记忆要点 | `#1976D2`（深蓝） | 核心记忆点/口诀 |

## 解析结构

```html
<div style="color:#2E7D32"><b>✓ 正确答案</b></div>
选项X：正确理由，关键依据

<div style="color:#D32F2F"><b>✗ 错误选项</b></div>
①选项Y：错误原因
②选项Z：易混淆点

<div style="color:#1976D2"><b>💡 记忆要点</b></div>
核心记忆点/口诀/对比
```

---

# 符号压缩

| 符号 | 含义 | 示例 |
|:---|:---|:---|
| → | 导致/流程 | 输入→处理→输出 |
| ↑↓ | 增减 | 效率↑、成本↓ |
| ①②③ | 并列 | ①特点A②特点B③特点C |
| `<u>下划线</u>` | 重点 | `<u>核心概念</u>` |
| `<b>加粗</b>` | 强调 | `<b>关键词</b>` |

# 完整示例

## 示例1：单选题

```
Python || 数据类型 || 以下哪个是不可变类型？	A列表list<br>B字典dict<br>C元组tuple<br>D集合set	C	<div style="color:#2E7D32"><b>✓ 正确答案</b></div>C元组tuple：创建后<u>不能修改</u>元素，是不可变类型<br><br><div style="color:#D32F2F"><b>✗ 错误选项</b></div>①A列表：可通过索引修改元素<br>②B字典：可增删改键值对<br>③D集合：可add/remove元素<br><br><div style="color:#1976D2"><b>💡 记忆要点</b></div>不可变：<b>数字、字符串、元组</b>；可变：列表、字典、集合
```

## 示例2：情境题

```
项目管理 || 风险管理 || 项目预算紧张，发现一个低概率高影响风险，应采取？	A立即投入资源消除<br>B制定应急计划备用<br>C忽略该风险<br>D转移给第三方	B	<div style="color:#2E7D32"><b>✓ 正确答案</b></div>B制定应急计划：低概率→不值得立即投入；高影响→不能忽略；应急计划成本低且有备无患<br><br><div style="color:#D32F2F"><b>✗ 错误选项</b></div>①A立即消除：预算紧张+低概率，投入产出比不合理<br>②C忽略：高影响风险不能忽略<br>③D转移：通常需要额外成本（保险费等）<br><br><div style="color:#1976D2"><b>💡 记忆要点</b></div>风险策略选择看<b>概率×影响</b>和<b>成本</b>
```

## 示例3：多选题

```
数据结构 || 排序算法 || 以下哪些是稳定排序算法？	A冒泡排序<br>B快速排序<br>C归并排序<br>D选择排序	AC	<div style="color:#2E7D32"><b>✓ 正确答案</b></div>A冒泡排序+C归并排序：相等元素的相对位置不变<br><br><div style="color:#D32F2F"><b>✗ 错误选项</b></div>①B快速排序：partition过程会打乱相等元素顺序<br>②D选择排序：每次选最小交换，会跳过相等元素<br><br><div style="color:#1976D2"><b>💡 记忆要点</b></div>稳定："<b>冒插归</b>"（冒泡、插入、归并）；不稳定："<b>快选堆希</b>"
```
